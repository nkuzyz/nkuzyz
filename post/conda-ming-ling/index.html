conda 命令

0. 命令分类

    主要就是 基于 Anaconda 的 虚拟环境、包 的相关操作
        用户手册： https://conda.io/projects/conda/en/latest/user-guide/index.html
    conda --help 
        conda -h 
    conda --version 
        conda -V
    所有 --单词 都可以用 -单词首字母来代替
        比如 -version 可以用 -V来代替，只不过有的是大写，有的可能是小写
    分类：
    1- 自身管理
    2- 环境管理 -- 虚拟环境管理
    3- 包管理   -- 类似 pip 命令
    说明： 
        一般来说 conda 仓库的软件没有 PyPI 更新快和全。所以推荐 conda 只用来创建虚拟环境。
        包的安装管理仍然使用 pip。
            在 windows 下由于依赖不好处理，在使用 pip 失败时可以查询是否有编译好的 conda 包并使用。
    前提：借助 Anaconda 的命令进入 字符界面 == prompt 
        默认进入 Anaconda 的基准虚拟环境-base

1. 自身管理

-- 获取相关信息
    conda --help 
        conda 命令 --help 
        如：
            conda 有很多命令，要查看具体命令帮助，同样使用 -h | --help 选项
            conda config --help 
    conda --version     返回Anaconda软件的版本
    conda info          # 显示 Anaconda 当前的相关信息
        conda info -e   # 列出 Anaconda 当前的所有的虚拟环境、当前活跃前有*
-- 更新命令集合
    conda update --all # 更新 Anaconda 当前虚拟环境中的所有的第三方模块
        conda upgrade --all # 同上
    conda update anaconda   # 更新 Anaconda 整体版本
    conda update conda      # 更新 conda 命令，Conda会检查可升级版本，并同时显示可升级的包。
    conda update python     # 更新 Anaconda当前的Python环境到支持的最新版本
    原因： Anaconda 将 anaconda、conda、python 都当成 第三方模块 来管理
-- 配置命令集合
    conda info  显示当前 conda 安装的所有信息。
        通常使用 -e/--envs 选项查看创建的虚拟环境
    conda config --show 查看配置文件中所有的配置信息， 关注其中的 key
        配置文件就是 conda config --set show_channel_urls yes 生成|修改的 .condarc 文件 
        其中的 key 可以 用于修改 配置文件 中的配置。
      如：查看、修改 通道(用于镜像的)
            为什么要做通道管理
                在国内用官方的通道速度会非常慢，所以一般都需要把通道换成国内的，
                国内有很多镜像源，像清华，中科大等，推荐使用清华源。
        # 记得，修改源后，要加上下面这句！目的：设置搜索时显示通道地址
        conda config --set show_channel_urls yes        设置搜索时显示通道地址
        -- channel 显示、恢复、添加、删除(可以使用 conda config 命令，本质是修改 .condarc 文件)
        # 显示现有安装源
        conda config --show channels        
        # 恢复默认源
        conda config --remove-key channels  
        # 更换|添加清华源
        conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/
        conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud/conda-forge 
        conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud/msys2/
        # 更换|添加中科大源
        conda config --add channels https://mirrors.ustc.edu.cn/anaconda/pkgs/main/
        conda config --add channels https://mirrors.ustc.edu.cn/anaconda/pkgs/free/
        conda config --add channels https://mirrors.ustc.edu.cn/anaconda/cloud/conda-forge/
        conda config --add channels https://mirrors.ustc.edu.cn/anaconda/cloud/msys2/
        conda config --add channels https://mirrors.ustc.edu.cn/anaconda/cloud/bioconda/
        conda config --add channels https://mirrors.ustc.edu.cn/anaconda/cloud/menpo/
        # 移除某个源
        conda config --remove channels https://mirrors.cloud.tencent.com/anaconda/pkgs/pro/

2. 环境管理 -- 虚拟环境管理

    虚拟环境的必要性： 
        安装了 Python3 但也要用 Python2 的，怎么办？不需要重新安装 Python...
        对于深度学习项目，如何配置不同的环境以匹配不同的框架（如 Pytorch，MXNet，TensorFlow）？
        如何将已经配置好的环境迁移到其他电脑上，而无需重新配置环境？
        如何直接在 Jupyter Notebook 中切换环境？
      均可借助 Anaconda 的虚拟环境管理来实现。
    Python 虚拟环境： 
        自行指定 虚拟环境的容器目录
        Python 的所有的虚拟环境 都是基于当前同一个 Python 的版本
    Anaconda 虚拟环境 
        1- 需要不同的虚拟环境，每个虚拟环境中的所需的 第三方模块 各不相同
        2- 需要不同的虚拟环境，每个虚拟环境中：
            所基于的 Python 的版本各不相同
            所需的 第三方模块 各不相同
        3- 需要将某个位置上的虚拟环境 复制(克隆、导出) 到另外的位置上
0. 虚拟环境 容器目录：
        Anaconda 默认提供虚拟环境 base，安装后，进入的就是 base 
            当前运行环境会显示在括号中，默认是 base
            最好不要在 base 环境中安装或者配置一些包之类的，
            它是 Python 的基础环境，为了更好的管理 Python 环境，需要创建属于自己的虚拟环境。
        可以使用：conda create --name <env_name> <package_names>
            也可以简写为 conda create -n <env_name> <package_names>。
            其中 <env_name> 与 <package_names> 分别表示创建的环境名以及环境中的包名。
            建议以英文命名，且不加空格，名称两边不加尖括号 <>。
        可以使用 conda create -n / --name 创建虚拟环境。
            默认的创建的位置，可以在 conda config --show 中的 envs_dirs: 看到。 
        修改 虚拟环境的位置：
            1- 借助 conda create 命令在创建的时候，借助参数(-p|--prefix)指定虚拟环境的文件夹的位置
                conda create -p 虚拟环境的位置 -n 虚拟环境的名称
             -p PATH, --prefix PATH
                Full path to environment location (i.e. prefix).
            2- 修改配置文件，指定默认的虚拟环境的容器目录位置
                conda config --show     查看虚拟环境安装位置
                修改配置语法：
                    conda config --add key value #添加语法
                    conda config --remove key value #删除语法
                        key为 envs_dirs、pkgs_dirs等
                        value为key对应的值
                添加、删除envs_dirs:
                    conda config --add envs_dirs dir
                    conda config --remove envs_dirs dir
                如：要用  / 
                    conda config --add envs_dirs c:/File/Anaconda3/envs
                    conda config --remove envs_dirs /Users/Lsz/.conda/env
1. 常用的虚拟环境的命令：（创建、查看、删除、复制、进入、退出）
        #创建环境
            1= conda create -n environment_name
            如： 
                # envname 指代新建的环境的名称
                conda create --name envname
                # 后面会让用户确认,输入y回车就好
                # 也可同时指定要加入的 第三方模块|包以及版本，多个以 空格 间隔
                conda create -n envname numpy scipy   
                conda create -n envname numpy=1.1 scipy=1.25   
            2= conda create -n envname python=3.7
                #创建指定python版本下(还可以同时指定包含某些包)的环境
                conda create -n environment_name python=3.7 numpy scipy
                如： 
                    # 指定python版本
                    conda create -n envname python=3.6
                    # 指定包
                    conda create -n envname scipy
                    # 指定包版本
                    conda create -n envname scipy=0.15.0
                    # 指定python版本,以及多个包
                    conda create -n envname python=3.6 scipy=0.15.0
                即：
                    如果要安装指定的版本号，则只需要在包名后面以 = 和版本号的形式执行。
                    如： conda create --name python2 python=2.7 
                        即创建一个名为 “python2” 的环境，环境中安装版本为 2.7 的 python。
                    如果要在新创建的环境中创建多个包，
                        则直接在 <package_names> 后以空格隔开，添加多个包名即可。
                    如： conda create -n python3 python=3.5 numpy pandas
                        即创建一个名为 “python3” 的环境，
                        环境中安装版本为 3.5 的 Python，同时也安装了 numpy 和 pandas。
        #删除环境
            conda remove -n yourname --all
            如： 
                加上参数 --all 并使用 -n/--name 指定要删除的环境名。
                conda remove --name envname --all
            --all 删除指定虚拟环境中的所有的第三方模块
              如果当前虚拟环境中没有安装任何第三方模块， --all 会同时删除虚拟环境所在的目录
              如果当前虚拟环境中有安装第三方模块， --all后需手工删除虚拟环境所在的目录
        #查看|列出环境
            conda env list / conda info -e / conda info --envs
            当前激活的运行环境有且只有一个，且前面有星号(*)
        #复制环境-克隆环境
            整体复制本机的另一个名称的虚拟环境中
            conda create --name new_env_name --clone old_env_name
            conda create --name <new_env_name> --clone <copied_env_name>
            如： 
                # clone_env 代指克隆得到的新环境的名称
                # envname 代指被克隆的环境的名称
                conda create --name clone_env --clone envname
        #复制-导入/导出
            # 生成虚拟环境的配置文件，并依照配置文件生成虚拟环境
                即： 导入导出环境
            #导出：到指定目录下生成环境yml文件 -- 导出当前虚拟环境的配置到 yml 文件
                conda env export > 目录/environment.yml
                将包信息存入 yaml 文件中。当需要重新创建一个相同的虚拟环境时可以用
            #导入：从yml文件创建环境 -- 可以在另一个位置|其他的计算机上 
                conda env create -n env_name -f environment.yml
        #进入|激活环境
            默认处于 base 环境，进入其他环境需要使用 activate 手动切换
            conda activate environment_name
                或者 直接： activate environment_name
            切换不同的 Anaconda 中的虚拟环境
            如：
                # for windows
                conda activate envname
                    或者： activate envname
                # for liunx and MACOS 
                source activate envname
        # 退出|关闭环境
            conda deactivate
                或者 直接： deactivate
            切换回原先的 虚拟环境
            如：
                # for windows
                conda deactivate
                    或者： deactivate
                # for linux and MACOS
                source deactivate
3. 包管理 -- 类似 pip ，也可以使用 pip
    0. 对包的管理是在某个环境下进行的
        首先要进入特定环境再进行包的操作
        而且，所有的命令，均有两个版本：
            对当前虚拟环境、对指定虚拟环境
    1. 常见的包管理的命令
        #安装包 -- (在某个虚拟环境中安装第三方模块)
            使用 conda install 安装 conda 包，会自动处理包之间的依赖。
            conda instal packagename packagename
            conda install -n envname packagename packagename
            如： 
                # 首先需要创建名称为envname的虚拟环境
                conda install -n envname scipy
                # 指定包版本conda install -n envname scipy=0.15.0
                # 向当前环境安装包 scipy
                conda install scipy
                # 向当前环境安装包 scipy 并指定版本
                conda install scipy=0.15.0
                # 向当前环境安装多个包
                conda install scipy curl
                # 向当前环境安装多个包,并指定包版本
                conda install scipy=0.15.0 curl=7.26.0
                # 向当前环境安装多个包,并指定包版本与python版本,一般情况下不用指定python版本
                conda install scipy=0.15.0 curl=7.26.0 -n py34_env
                # 向指定虚拟环境中安装指定包
                conda install --name myenv scipy
            使用 pip 安装包
                使用场景：当使用 conda install 无法进行安装时，可以使用 pip 进行安装。
                pip 只是包管理器，无法对环境进行管理。
                因此如果想在指定环境中使用 pip 进行安装包，则需要先切换到指定环境中，再使用 pip 命令安装包。
                pip 无法更新 Python，因为 pip 并不将 python 视为包。
                pip 可以安装一些 conda 无法安装的包；conda 也可以安装一些 pip 无法安装的包。
                因此当使用一种命令无法安装包时，可以尝试用另一种命令。
              比如： scikit-learn、tensorflow 建议使用 pip install 安装 
                可以安装最新版，可以支持 GPU
        #查看(列出某个虚拟环境中所有第三方模块)
            #查看当前环境包列表
            conda list
            #查看指定环境包列表
            conda list -n environment_name
            如： 
                # 查看当前换件中安装的所有包
                conda list
                # 查看指定虚拟环境中安装的所有包
                conda list -n envname
                # 可以使用如下命令安装pip到当前虚拟环境
                conda install pip
                # 使用pip安装numpy
                pip install numpy
        #搜索包：查看可用的conda源中包的信息
            #python pip 命令
            pypi.org 上去检索所有可用的 第三方模块
            #使用 conda search 查询 conda 包
                Anaconda 所有的可用的 第三方模块 是 Anaconda 自己维护
            conda search package_name
            如： 
                # 在所有添加的仓库以及默认仓库中搜索包 numpy
                conda search numpy
            只有经过 conda 重新编译入库的才能使用 ，conda search 查询得到
            再具体的可以进 Anaconda package lists 查看。
          -- 详细讲解： 分为精确查找和模糊查找
            精确查找
                conda search --full-name <package_full_name>
                --full-name 为精确查找的参数。<package_full_name>是被查找包的全名。
            模糊查找
                conda search <text>
                其中 <text> 是查找含有此字段的包名。
              例如：conda search py 即查找含有 py 字段的包，有哪些版本可供安装。
          -- Anaconda 自己维护 第三方模块 的版本 可能不是最新 
            会有一个滞后，也是在 Anaconda 环境中有时必须使用 pip 的原因
        #更新包(更新某个虚拟环境中所有第三方模块)
            更新当前环境中的指定包
                conda update package_name
                更新多个指定包，则包名以空格隔开，向后排列。
                如：conda update pandas numpy matplotlib 即更新 pandas、numpy、matplotlib 包。
            更新当前环境中的所有包
                conda update --all
                建议：在安装 Anaconda 之后执行上述命令更新 Anaconda 中的所有包至最新版本，便于使用。
            更新指定环境
                conda update -n environment_name ...
        #删除包
            conda remove package_name
            也可使用 conda uninstall
            如：
                # 删除当前环境中的scipy包
                conda remove scipy
                # 删除当前环境中的多个包
                conda remove scipy curl
                # 删除指定环境中的指定包
                conda remove -n myenv scipy
        #清理无用的安装包
            conda clean -p
            清空缓存:
            conda clean -i
        #清理tar包
            conda clean -t
        #清理所有安装包及cache
            conda clean -y --all
